Lab 8.1.2 - SYN Flood DoS Attack

CYBER.ORG Range Worksheet

Objective

Demonstrate how a Denial of Service attack affects availability of services and data.

Instructions:

  • Go to CYBER.ORG Range (https://apps.cyber.org)
  • Open the Kali Linux and Windows 7 environments
  • Select a partner for this lab. Select one person to be ‘STUDENT1’ and the other to be ‘STUDENT2’.

Part 1 - Normal web connection

1. STUDENT1 and STUDENT2, both open terminals and identify IP addresses for their Kali VMs.
2. STUDENT2 on your Kali VM in the terminal view the current network connections by typing: netstat -atn | less (press the space bar to page through the results).

This is the Cyber.org Range server IP address and the connection makes it possible for you to access the VM from your PC.

Click here and press Ctrl + V (or Cmd + V) to paste your screenshot.

4. On STUDENT2 Kali, press the Q key to exit netstat.

Part 2 - SYN FLOOD

The image below gives an overview of how the Attacker will flood the Victim with so many SYN packets that the victim’s resources are overwhelmed, and it eventually crashes, making it unavailable to the user or any connected PCs.

Observe a SYN flood of packets from STUDENT1 Kali to STUDENT2 Kali
1. Student1 open a terminal on your Kali VM. Use the hping3 tool to send a SYN flood to the STUDENT2 Kali device by entering this command: 
sudo hping3 --flood -S -p 22 <S2 Kali IP Address>
Command translation:
--flood = send packets at the attacker computer’s top speed.
-S = SYN flag set in the packet.
-p 22 = send packets to this port number

2. STUDENT2, on your Kali VM open a terminal and start netstat again but this time add the -c for continuous mode: netstat -atn -c

Click here and press Ctrl + V (or Cmd + V) to paste your screenshot.

3. STUDENT1 and STUDENT2, stop the activity by entering CTRL-C in each Kali terminal.

Part 3 - Denial of Service (DoS)

1. STUDENT2 switch to your Windows7 VM.
2. Install the Clock Gadget (Start | Clock | Add to desktop). Double click to select Clock. Close Gadget Gallery.
3. Right-click on the Clock and select “Options.”
4. KEEP the first clock type. Check “Show the Second Hand”. Click “OK.”
We will use the second hand on the clock to identify when the Win7 system is slowing down or freezing from the SYN Flood.

5. STUDENT1 in the terminal on your Kali VM, use the hping3 tool to send a SYN flood with large packets to the STUDENT2 Windows device by entering this NEW hping3 command: 
sudo hping3 --flood -d 250 -S -p 80 <S2 Win7 IP Address>
Command translation:
-d 250 = packet size 250 data bytes
-p 80 = send packets to this port number

6. Student2, look at the Win7 Clock and notice that the second hand is stuttering and sometimes stopping. Try clicking on the Start Button or open Chrome – these will be unresponsive as the system freezes.
7. Student 1 stop the SYN Flood activity by entering CTRL-C in your Kali terminal.
